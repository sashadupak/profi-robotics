## Слайд 4 — PID-регулятор

### Текст
**PID-регулятор**

**Плюсы:**
- Прост в реализации
- Не требует измерения параметров системы

**Минусы:**
- Использует программное дифференцирование
- Необходим подбор коэффициентов

### Изображения

![Simulink: схема PID-регулятора](image4.png)
*Simulink-модель ПИД-подсистемы (Mobile_Tracking_PID2/Subsystem6): вход e, коэффициенты 2.5 / 0.01 / 0.5, интегратор 1/s, блок Δu/Δt, выход u*

![Страница Википедии — ПИД-регулятор с формулами](image5.png)
*Скриншот Википедии с базовыми формулами ПИД и блок-схемой управления*

### Формулы (из изображения image5)

$$u(t) = P + I + D = K_p e(t) + K_i \int_0^t e(\tau)\,d\tau + K_d \frac{de}{dt}$$

$$u(t) = K_p \left( e(t) + K_{ip} \int_0^t e(\tau)\,d\tau + K_{dp} \frac{de}{dt} \right)$$

$$U(n) = K_p E(n) + K_p K_{ip} T \sum_{k=0}^{n} E(k) + \frac{K_p K_{dp}}{T}(E(n) - E(n-1))$$

## Слайд 5 — LQR-регулятор

### Текст
**LQR-регулятор**

**Плюсы:**
- Оптимальное управление
- Возможность выбора критериев регулирования

**Минусы:**
- Только для линейных систем
- Требует точной идентификации системы

### Изображения

![LQR — страница Википедии с математическим выводом](image6.png)
*Finite-horizon, continuous-time LQR: уравнения состояния, функционал качества, Riccati ODE*

![LQR Block Diagram](image7.png)
*Блок-схема LQR: r → ⊗ → Plant (ẋ=Ax+Bu, y=Cx+Du) → x → y; обратная связь через –K*

### Формулы (из изображения image6)

Система:
$$\dot{x} = Ax + Bu$$

Функционал качества:
$$J = x^T(t_1)F(t_1)x(t_1) + \int_{t_0}^{t_1}(x^T Q x + u^T R u + 2x^T N u)\,dt$$

Оптимальный закон управления:
$$u = -Kx, \quad K = R^{-1}(B^T P(t) + N^T)$$

Уравнение Риккати:
$$A^T P(t) + P(t)A - (P(t)B + N)R^{-1}(B^T P(t) + N^T) + Q = -\dot{P}(t)$$

## Слайд 6 — Метод функций Ляпунова

### Текст
**Метод функций Ляпунова**

**Плюсы:**
- Функция исключает увеличение ошибки
- Не требуется полная модель системы

**Минусы:**
- Недостаточная величина управления при малых отклонениях

### Изображения

![Формулы условия устойчивости Ляпунова](image8.png)
*Условия: V(x₁,x₂) > 0, V̇(x₁,x₂) < 0; следствие: grad V · ẋ < 0 ⟹ lim x(t) = 0 при t→∞*

![3D-визуализация функции Ляпунова](image9.jpeg)
*Чаша-параболоид V(x₁, x₂) с векторами grad V и dX/dt; точка A — начальное состояние, O — точка равновесия*

### Формулы (из изображения image8)

$$V(x_1, x_2) > 0, \quad \dot{V}(x_1, x_2) < 0$$

$$\text{grad}\,V \cdot \dot{x} < 0 \implies \lim_{t \to \infty} x(t) = 0$$

## Слайд 8 — Теория | Робастное управление

### Текст
**Теория | Робастное управление**

Рассмотрим робастный закон управления, основанный на принципах сильной обратной связи

### Изображения

![Simulink: схема робастного регулятора](image11.png)
*Simulink-модель (Mobile_Tracking_Robast2/Subsystem2): e → (+_) → ×8 → 1/s → (+) → ×3 → u; ветка 1/2 подаётся на сумматор*

![Слайд с формулами робастного закона](image12.png)
*Оригинальный слайд лекций ITMO University с выводом робастного закона управления*

### Формулы (из изображения image12)

$$u_i = \mu_i(\dot{\xi}_i + 2\xi_i)$$
$$\dot{\xi}_i = \sigma_i(-\xi_i + e_i)$$

где:
- $U_i$, $i = 1\ldots6$ — сигналы управления
- $e_i = \{z^* - z,\; \theta^* - \theta,\; \psi^* - \psi,\; \phi^* - \phi,\; \bar{x},\; \bar{y}\}$ — сигналы ошибок
- $\mu_i, \sigma_i$, $i = 1\ldots6$ — настраиваемые коэффициенты

Дополнительные условия:
$$\theta^* = \frac{U_5}{U_1}, \quad \psi^* = -\frac{U_6}{U_1}$$

Если $U_1$ близко к нулю, то $\theta^*, \psi^*$ равны нулю. Также $\theta^*, \psi^*$ не должны быть более $\pi/6$.

## Слайд 10 — Теория | Проекционное отображение

### Изображения

![Формулы матриц поворота и матрицы камеры](image15.png)
*Матрицы поворота R_x (по α) и R_y (по β), их произведение R = R_x R_y; вектор смещения p, матрица преобразования T, матрица камеры M и её псевдообратная M⁺; гомография H = MTM⁺*

![Калибровочная шахматная доска](image16.png)
*Фото шахматной доски для калибровки камеры (вид под углом)*

### Формулы (из изображения image15)

$$R_x = \begin{bmatrix}1 & 0 & 0 \\ 0 & \cos\alpha & -\sin\alpha \\ 0 & \sin\alpha & \cos\alpha\end{bmatrix}, \quad R_y = \begin{bmatrix}\cos\beta & 0 & \sin\beta \\ 0 & 1 & 0 \\ -\sin\beta & 0 & \cos\beta\end{bmatrix}$$

$$R = R_x R_y$$

$$p = \begin{bmatrix}\tan\alpha \\ \tan\beta \\ 1\end{bmatrix}, \quad T = \begin{bmatrix}R & p \\ 0 & 1\end{bmatrix}, \quad M = \begin{bmatrix}f_x & 0 & c_x & 0 \\ 0 & f_y & c_y & 0 \\ 0 & 0 & 1 & 0\end{bmatrix}$$

$$M^+ = \begin{bmatrix}1/f_x & 0 & -c_x/f_x \\ 0 & 1/f_y & -c_y/f_y \\ 0 & 0 & 0 \\ 0 & 0 & 1\end{bmatrix}$$

$$H = M T M^+$$

## Слайд 14 — Поиск ближайшей точки

### Текст
**Программная реализация — Поиск ближайшей точки на траектории**

### Код

```python
def distance(self, p1, p2):
    return sqrt((p1[0] - p2[0])**2 + (p1[1] - p2[1])**2)

def closest_point(self, point, array):
    closest = None
    min_val = float('inf')
    for i in range(len(array)):
        if (self.distance(point, array[i]) < min_val):
            min_val = self.distance(point, array[i])
            closest = array[i]
    return closest
```

## Слайд 15 — Низкочастотный фильтр вибраций

### Текст
**Программная реализация — Низкочастотный фильтр вибраций**

### Изображения

![Вывод дискретного фильтра НЧ](image24.png)
*Полный вывод: от передаточной функции H(s) = 1/(Ts+1) через образ Лапласа к дискретной реализации*

![АЧХ фильтра низких частот (1-й и 2-й порядок)](image25.gif)
*Частотная характеристика: Pass Band до fc, Stop Band после fc; наклон –20 дБ/декада (1-й порядок) и –40 дБ/декада (2-й порядок)*

### Формулы (из изображения image24)

Передаточная функция:
$$H(s) = \frac{y(s)}{u(s)} = \frac{1}{Ts + 1}$$

Дифференциальное уравнение:
$$T\dot{y}(t) + y(t) = u(t)$$

Дискретизация (метод Эйлера):
$$T\frac{y(t_k) - y(t_{k-1})}{h} + y(t_k) = u(t_k)$$

Итоговая дискретная формула:
$$y(t_k) = \frac{T}{T+h}\,y(t_{k-1}) + \frac{h}{T+h}\,u(t_k)$$

$$\boxed{y(t_k) = (1-\alpha)\,y(t_{k-1}) + \alpha\,u(t_k)}, \quad \alpha = \frac{h}{T+h}$$
